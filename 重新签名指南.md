# 应用重新签名指南

由于修改了包名（Bundle Identifier）从 `io.appflowy.appflowy` / `com.appflowy.appflowy.flutter` 改为 `ioi.xiaomabiji.ponynotes`，需要重新配置签名。

## iOS 重新签名步骤

### 1. 在 Xcode 中配置签名

1. 打开 Xcode 项目：
   ```bash
   open frontend/appflowy_flutter/ios/Runner.xcworkspace
   ```

2. 在 Xcode 中：
   - 选择 `Runner` target
   - 进入 `Signing & Capabilities` 标签
   - 确保 `Bundle Identifier` 为 `ioi.xiaomabiji.ponynotes`
   - 选择你的 `Team`（开发团队）
   - Xcode 会自动生成新的 Provisioning Profile

### 2. 如果使用自动签名（推荐）

- 勾选 `Automatically manage signing`
- 选择你的 `Team`
- Xcode 会自动处理 Provisioning Profile

### 3. 如果使用手动签名

- 取消勾选 `Automatically manage signing`
- 在 Apple Developer Portal 创建新的 App ID: `ioi.xiaomabiji.ponynotes`
- 创建新的 Provisioning Profile 并下载
- 在 Xcode 中选择该 Provisioning Profile

### 4. 清理并重新构建

```bash
cd frontend/appflowy_flutter
flutter clean
cd ios
pod install
cd ..
flutter build ios
```

## macOS 重新签名步骤

### 1. 在 Xcode 中配置签名

1. 打开 Xcode 项目：
   ```bash
   open frontend/appflowy_flutter/macos/Runner.xcworkspace
   ```

2. 在 Xcode 中：
   - 选择 `Runner` target
   - 进入 `Signing & Capabilities` 标签
   - 确保 `Bundle Identifier` 为 `ioi.xiaomabiji.ponynotes`
   - 选择你的 `Team`
   - 勾选 `Automatically manage signing`

### 2. 清理并重新构建

```bash
cd frontend/appflowy_flutter
flutter clean
cd macos
pod install
cd ..
flutter build macos
```

### 3. 如果运行后提示 HotKey.framework 签名不匹配

修改包名后，旧的框架仍然保留了原签名，系统会以
`code signature ... have different Team IDs` 为理由拒绝加载。
构建完成后请执行以下命令重新为所有内嵌框架签名：

```bash
APP_PATH="frontend/appflowy_flutter/build/macos/Build/Products/Release/PonyNotes.app"
# 使用自己的签名证书名称，例如 "Apple Development: Your Name (TEAMID)"
SIGN_IDENTITY="Apple Development: Your Name (TEAMID)"

# 递归为所有 Framework / Plugin / 辅助二进制重新签名
codesign --force --deep --options runtime --sign "$SIGN_IDENTITY" "$APP_PATH"

# 如果你只想临时以 ad-hoc 方式签名（不开启 Gatekeeper 验证），可以把 SIGN_IDENTITY 改成 "-"
# codesign --force --deep --sign - "$APP_PATH"
```

完成后再次运行：

```bash
open "$APP_PATH"
```

若仍有崩溃，可使用 `codesign -vvv` 检查具体哪一个文件未被正确签名。

## Android 重新签名

Android 应用在修改包名后需要：

1. 清理构建：
   ```bash
   cd frontend/appflowy_flutter
   flutter clean
   ```

2. 重新构建：
   ```bash
   flutter build apk
   # 或
   flutter build appbundle
   ```

3. 如果使用签名配置，确保 `key.properties` 文件中的配置正确

## 常见问题

### 问题 1: "No profiles for 'ioi.xiaomabiji.ponynotes' were found"

**解决方案：**
- 在 Apple Developer Portal 创建新的 App ID
- 或者使用自动签名让 Xcode 自动创建

### 问题 2: "Signing for "Runner" requires a development team"

**解决方案：**
- 在 Xcode 的 Signing & Capabilities 中选择你的开发团队
- 确保你的 Apple ID 已添加到 Xcode 的 Accounts 中

### 问题 3: 应用无法安装或打开

**解决方案：**
1. 删除设备上旧版本的应用
2. 清理构建缓存：`flutter clean`
3. 重新构建并安装

### 问题 4: CocoaPods 编码错误

**解决方案：**
```bash
export LANG=en_US.UTF-8
cd frontend/appflowy_flutter/ios
pod install
```
